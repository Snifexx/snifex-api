A sized string view. 

This a simple sized string view... Nothing else. For the uninitiated\+:
\begin{DoxyItemize}
\item \textquotesingle{}sized\textquotesingle{} means that we carry the length of the data with the data (the char\texorpdfstring{$\ast$}{*})
\item \textquotesingle{}view\textquotesingle{} because the memory is not owned. That means that it is the user responsibility to manage the memory the string view is pointing to. The recommended way is using an arena.
\end{DoxyItemize}

\begin{DoxyNote}{Note}
\doxylink{group__string}{Strings} are NOT zero tailing (since they\textquotesingle{}re string-\/views anyways) 
\end{DoxyNote}

Returns result of concatination of string {\ttfamily b} into {\ttfamily a} 

\begin{DoxyParagraph}{Implementation details}
If we are not careful we could execute memcpy(ptr, NULL, 0), where ptr is guaranteed not to be NULL, which is U.\+B. before c2y. Clang and GCC apparently handle this gratefully while MSVC does not. So to conform to c99 we check for 0-\/len before memcpy-\/ing. This isn\textquotesingle{}t not really a performance issue in release mode as it should just be optimised out (and gcc/clang with -\/O1 surely does)
\end{DoxyParagraph}
