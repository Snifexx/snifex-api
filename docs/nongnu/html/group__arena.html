Detailed Description

Fixed-sized and growing arenas.

My implementation of an Arena allocator. I've implemented both a fixed size Arena, and a dynamically growing one.

dyn_arena,

Get a temporary pointer from relative pointer of a DynArena.

Get a pointer to the object at from rel_ptr. The returning pointer should not be stored or reused after a dyn_arena_alloc allocation, because an allocation could trigger a resizing rendering that pointer dangerous. Instead, store, pass and work with the relative pointer retuned from dyn_arena_alloc and then, when working on the object in the DynArena, get a temporary "absolute" pointer with this macro

Parameters
tThe type of the item we're casting the pointer to
rel_ptrThe relative pointer to the object in the DynArena
Returns
An "absolute" pointer to the object of type t allocated in the DynArena at offset rel_ptr

Typedef Documentation

◆ Arena

typedef struct arena Arena

A fixed-sized arena.

A fixed-sized arena, that allocates a certain amount of memory when initialized and never gets reallocated again. Because of this allocations can fail when there is not enough memory allocated. Since reallocations never happen, arena_alloc return pointers, which is VERY useful for strings or scratch arenas

◆ DynArena

typedef struct dyn_arena DynArena

A dinamically growing arena.

An arena that is not fixed-sized, and dynamically grows. Because of this property, dyn_arena_alloc does not return a pointer, it returns an index with which we can access the pointer temporarily (with dyn_arena_get)

extern

Reserves at least min_cap bytes in the arena.

Allocates enough bytes of memory for the DynArena capacity to be at least min_cap. It is not a guaranteed realloc: it COULD trigger one, but only if dyn_arena->cap < min_cap.

Postcondition
dyn_arena->buf != NULL if malloc did not fail
