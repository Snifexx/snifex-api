Detailed Description

A sized string view.

This a simple sized string view... Nothing else. For the uninitiated:

  • 'sized' means that we carry the length of the data with the data (the char*)
  • 'view' because the memory is not owned. That means that it is the user responsibility to manage the memory the string view is pointing to. The recommended way is using an arena.
Note
Strings are NOT zero tailing (since they're string-views anyways)
